**B. Create `src/components/quiz/step-progress.tsx`** — "Paso X/3" + segmented bar:
- Props: `currentStep: number` (0-indexed)
- Display "Paso {currentStep + 1}/3" in text-small text-muted-foreground
- 3-segment bar: each segment is `h-1 flex-1 rounded-full`, separated by `gap-1`
- Active segments (i <= currentStep): `bg-primary`, inactive: `bg-border`
- Segment transition: `transition-colors duration-200`
- `role="progressbar"` with `aria-valuenow={currentStep + 1}`, `aria-valuemin={1}`, `aria-valuemax={3}`, `aria-label="Progreso del quiz"`

**C. Create `src/components/quiz/option-card.tsx`**:
- Props: `option: QuizOption`, `isCenter: boolean`, `isSelected: boolean`, `onClick: () => void`
- Renders as a `<button>` with `role="option"` and `aria-pressed={isSelected}`
- Card dimensions: `w-[280px] md:w-[320px]` (fixed width, no shrink)
- Card style: `rounded-2xl border-2 p-4 flex flex-col items-center gap-3`
- Selected (isCenter) state: `border-primary ring-2 ring-primary/20` — applied when `isCenter` is true
- Unselected state: `border-border`
- Focus visible: `focus-visible:ring-2 focus-visible:ring-primary`
- Tap target: entire card is the button (well over 44px)
- Content: illustration component (80x80 display, from ILLUSTRATIONS map) centered, then label below in text-body
- Do NOT apply Framer Motion scale here — the parent carousel handles scale animation via `motion.div`

**D. Create `src/components/quiz/option-carousel.tsx`** — Framer Motion carousel:
- Props: `options: QuizOption[]`, `selectedValue: string | null`, `onSelect: (value: string) => void`
- Track `centerIndex`: derive from `selectedValue` position in options array. If null, default to visual center (index 1) but do NOT write to state — only user interaction writes to state
- **Arrow buttons:** Two buttons, positioned absolute left-0 and right-0, vertically centered on the card row. Size: `w-11 h-11` (44px touch target). Icons: `ChevronLeft` and `ChevronRight` from lucide-react, size-5. `aria-label="Opcion anterior"` and `aria-label="Opcion siguiente"`. Disable left arrow when centerIndex=0, disable right arrow when centerIndex=2 (opacity-30 + pointer-events-none when disabled).
- **Drag container:** `motion.div` with `drag="x"`, `dragConstraints={{ left: 0, right: 0 }}`, `dragElastic={0.15}`, `onDragEnd` handler: if offset.x < -40 move right (centerIndex + 1), if offset.x > 40 move left (centerIndex - 1). Snap transition: `type: "spring", stiffness: 300, damping: 30`.
- **Card rendering:** 3 `motion.div` wrappers, each containing `<OptionCard>`. The `motion.div` wrapper applies `animate={{ scale: isCenter ? 1.03 : 0.92, opacity: isCenter ? 1 : 0.6 }}` with `transition={{ duration: 0.2, ease: "easeOut" }}`.
- Layout: `flex items-center justify-center` with horizontal overflow visible. Cards have `shrink-0`. Non-center cards peek from edges.
- On arrow click: `onSelect(options[newIndex].value)` — this triggers parent state update which re-derives centerIndex.

**E. Create `src/components/quiz/quiz-step.tsx`** — Single step layout:
- Props: `stepIndex: number`, `stepData: QuizStepDef`, `currentSelection: string | null`, `onSelect: (value: string) => void`, `onNext: () => void`, `onBack: () => void`
- Layout (top to bottom):
  1. Back arrow: top-left, `ChevronLeft` icon in a 44x44 button. Hidden on step 0 (`stepIndex === 0`). `aria-label="Volver al paso anterior"`. Calls `onBack`.
  2. Step heading: `text-subhead font-medium text-foreground text-center` — renders `stepData.heading`
  3. StepProgress component: `currentStep={stepIndex}`
  4. OptionCarousel: `options={stepData.options}`, `selectedValue={currentSelection}`, `onSelect={onSelect}`
  5. "Siguiente" button: full-width, primary variant, `disabled={!currentSelection}`. Text: "Siguiente →" for steps 0-1, "Ver mis recomendaciones →" for step 2. Calls `onNext`.
  6. "Volver al Inicio" link: `text-body text-muted-foreground hover:text-foreground text-center` — links to "/" via Next.js Link.

**F. Create `src/components/quiz/quiz-shell.tsx`** — Step state machine:
- "use client" directive at top
- State: `currentStep` (0-2), `selections` as `[string | null, string | null, string | null]`, `direction` (1 or -1), `quizComplete` (boolean), `profileData` (null initially)
- localStorage sync: on mount, read `QUIZ_STORAGE_KEY` from localStorage and restore `selections` if found. On every selection change, write `{ selections }` to localStorage.
- CRITICAL: wrap ALL localStorage reads in useEffect (SSR safety — see RESEARCH.md Pitfall 1)
- handleNext: set direction=1, if currentStep===2 set quizComplete=true, else increment currentStep. Persist to localStorage.
- handleBack: set direction=-1, decrement currentStep (min 0).
- handleSelect: update selections[currentStep] to new value.
- When quizComplete transitions to true: call `fetchProfile(selections[0] as Workload, selections[1] as Lifestyle, selections[2] as Budget)` then `fetchLaptopsByIds(profile.laptop_ids)`. Store result in profileData. Also write completed profile to `PROFILE_STORAGE_KEY` in localStorage (for Navbar avatar in Plan 04).
- Rendering:
  - If quizComplete and profileData loaded: render `<QuizResult>` (imported but implemented in Plan 03 — for now import conditionally or render a placeholder div with data-testid="quiz-result-placeholder" and the profile name)
  - If quizComplete and loading: render a loading indicator (can be simple "Cargando..." text for now — Plan 03 adds the full skeleton)
  - Otherwise: render the step carousel inside AnimatePresence:
    ```tsx
    <AnimatePresence mode="wait" custom={direction}>
      <motion.div
        key={currentStep}
        custom={direction}
        variants={{
          enter: (dir: number) => ({ x: `${dir * 60}%`, opacity: 0 }),
          center: { x: 0, opacity: 1 },
          exit: (dir: number) => ({ x: `${dir * -60}%`, opacity: 0 }),
        }}
        initial="enter"
        animate="center"
        exit="exit"
        transition={{ duration: 0.3, ease: "easeInOut" }}
      >
        <QuizStep
          stepIndex={currentStep}
          stepData={QUIZ_STEPS[currentStep]}
          currentSelection={selections[currentStep]}
          onSelect={handleSelect}
          onNext={handleNext}
          onBack={handleBack}
        />
      </motion.div>
    </AnimatePresence>
    ```
- Import `fetchProfile` and `fetchLaptopsByIds` from `@/lib/quiz-data`.
- Import `QUIZ_STEPS`, `QUIZ_STORAGE_KEY`, `PROFILE_STORAGE_KEY`, `Workload`, `Lifestyle`, `Budget` from `@/types/quiz`.
