   ```typescript
   interface CatalogFilters {
     brands: string[];

     priceMax: number | null;
     screenSizes: string[];
     weights: string[];
     usageProfiles: UsageProfile[];
   ```

   **Data fetching** — in `useEffect` on mount:
   - Call `fetchAllLaptops()`, set `laptops`, set `loading = false`
   - On error: set `error` to `"No pudimos cargar los laptops. Verifica tu conexion e intenta de nuevo."`
   - Read `localStorage.getItem(PROFILE_STORAGE_KEY)`, parse JSON, set `completedProfile` if valid

   **Filtering** — `useMemo` deriving `filteredLaptops` from `laptops + searchText + filters + profileFilter`:
   - If `profileFilter` is true and `completedProfile` exists: filter to only laptops whose `id` is in `completedProfile.laptop_ids`
   - Text search: filter by `name.toLowerCase().includes(q)`, `brand.toLowerCase().includes(q)`, or any `simplified_tags` element containing q
   - Brand filter: `filters.brands.includes(l.brand)`
   - Price range: `l.price >= filters.priceMin` and `l.price <= filters.priceMax` when not null
   - OS filter: `filters.os.includes(l.os)` when `l.os` is not null
   - Screen size filter: `filters.screenSizes.includes(l.screen_size)` when not null
   - Weight filter: `filters.weights.includes(l.weight)` when not null
   - Usage profile filter: `l.usage_profiles.some(p => filters.usageProfiles.includes(p))`

   **Pagination** — `PAGE_SIZE = 100`. `visibleLaptops = filteredLaptops.slice(0, page * PAGE_SIZE)`. "Cargar mas" button shows when `visibleLaptops.length < filteredLaptops.length`.

   **Search bar debounce** — use a `useRef` timer with 200ms debounce. The input updates a local `inputValue` state immediately (for responsiveness), and sets `searchText` after 200ms pause.

   **Overlay URL param** — read `searchParams.get("laptop")` to get `activeLaptopId`. Find `activeLaptop` from `laptops` array. This will be used by Plan 03 to render the DetailOverlay. For now, just compute `activeLaptopId` and define `handleVerMas` and `handleCloseOverlay`:
