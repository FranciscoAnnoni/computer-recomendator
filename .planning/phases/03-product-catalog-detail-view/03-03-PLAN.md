   ```tsx
   <div className="sticky top-0 z-10 flex items-center justify-between px-4 py-3 bg-background/80 backdrop-blur">
     <button

       aria-label="Cerrar"

       href={laptop.affiliate_link}

     </a>
   </div>
   ```

   At the end of the `<main>` element (AFTER the FilterDrawer, BEFORE closing `</main>`), add:
   ```tsx
   <AnimatePresence>
     {activeLaptop && (
       <motion.div
         key={activeLaptop.id}
         initial={{ y: "100%" }}
         animate={{ y: 0 }}
         exit={{ y: "100%" }}
         transition={{ duration: 0.3, ease: [0.32, 0.72, 0, 1] }}
         className="fixed inset-0 z-50 bg-background overflow-y-auto"
       >
         <DetailOverlay laptop={activeLaptop} onClose={handleCloseOverlay} />
       </motion.div>
     )}
   </AnimatePresence>
   ```

   CRITICAL per RESEARCH.md Pitfall 1: The `{activeLaptop && <motion.div ...>}` conditional MUST be a direct child of `<AnimatePresence>`. Do NOT wrap it in any extra div. The `key={activeLaptop.id}` is required for exit animations.

   Exit transition should use a slightly different easing:
   Add to the motion.div: no separate exit prop needed — Framer Motion uses `exit={{ y: "100%" }}` with the same transition by default. For a different exit easing per UI-SPEC (250ms, easeIn), update transition to use `transition` prop on the animate/exit separately if needed, OR use a single transition config: `transition={{ duration: 0.3, ease: [0.32, 0.72, 0, 1] }}`. The slight difference (300ms vs 250ms) is acceptable for v1.
